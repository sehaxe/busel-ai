## What is Busel?

Busel (Бусел, from Belarusian *бусел* — stork) is a from-scratch
implementation of a 1.58-bit LLM. It is small enough to read in an
afternoon (~3 000 LoC of Python + ~140 LoC of Rust), runs on a single
RTX 5060 Ti or Apple Silicon Mac, and includes a custom architecture that
combines several recent research ideas in one place.

<CardGrid stencil>
  <Card title="1.58-bit weights" icon="pencil">
    Every linear in the backbone quantises to `{-1, 0, +1}`. The Shpak
    profile (52.8 M params) ships as an **11 MB** checkpoint. Inference
    becomes pure additions on CPU.
  </Card>
  <Card title="mAR residuals" icon="document">
    Replace the classical `y = x + f(x)` with an input-dependent,
    doubly-stochastic mixture of the last `n_hyper` layer outputs,
    projected onto the **Birkhoff polytope** via Sinkhorn-Knopp.
    Combines [Kimi AttnRes](https://arxiv.org/abs/2603.15031) and
    [DeepSeek mHC](https://arxiv.org/abs/2512.24880).
  </Card>
  <Card title="3:1 GDN-2 / MLA / NSA" icon="setting">
    75 % of layers are linear (GDN-2, O(1) cache), 25 % are MLA
    (latent KV, `d_c=128`). NSA (Native Sparse Attention) provides
    dynamic sparse attention patterns. A 128 K-token context fits
    in ~98 MB of cache. SCT rank-8 compresses FFN weights 4-8×.
  </Card>
  <Card title="Byte-level (no BPE)" icon="star">
    Vocabulary of 277 tokens. The same byte stream carries text, code,
    JSON, images, and PDFs. The ByteFlow patcher compresses
    4 bytes → 1 patch with adaptive pooling and boundary detection.
  </Card>
  <Card title="SF-NorLotusMuon + FP8 AdamW" icon="rocket">
    2D weights go through SF-NorLotusMuon (Schedule-Free + NorMuon +
    LOTUS rank-8 factorised Muon, ~85× less optimizer state) with Gram
    Newton-Schulz orthogonalization; the rest goes through FP8 AdamW
    (`torchao.optim.AdamWFp8`, 75% memory reduction). Fused BitLinear
    Triton kernel in eager mode replaces 10+ ops in one launch. SCT
    rank-8, DropBP, RHO-Loss, Dispersion Loss, Progressive Freeze, and
    ASCII Curriculum are all ON by default. AutoPilot v6.0 handles
    spike recovery, adaptive clipping, and dynamic weight decay.
  </Card>
  <Card title="Plug-in registry" icon="puzzle">
    Adding a new attention or optimizer? Decorate it with
    `@register("kind", "name")` and the rest of the codebase picks it up
    automatically. No central switch statement to edit.
  </Card>
</CardGrid>
